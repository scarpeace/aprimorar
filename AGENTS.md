# Aprimorar Agent Notes

## Layout
- Real app roots are `server/` and `client/`. The root `package.json` is only helper scripts.
- `npm run start:backend` and therefore root `npm run dev` are currently stale: they point at nonexistent `server/api-aprimorar`. Run backend commands from `server/` directly.

## OpenSpec
- Read the relevant spec in `openspec/specs/` before changing behavior.
- Current specs are `parent-detail-dashboard` and `placeholder-feature-shells`.
- `openspec/changes/` currently only contains `archive/`; there is no active change folder to implement against.

## Backend
- Stack: Java 21, Spring Boot 3.5, Spring Modulith, PostgreSQL, Flyway.
- Start local DB from `server/` with `docker compose up -d db`.
- Main backend commands from `server/`:
  - `./mvnw spring-boot:run`
  - `./mvnw clean compile`
  - `./mvnw test`
  - Single test: `./mvnw test -Dtest=ClassName`
- `application.yml` keeps the `dev` profile active by default.
- `application-dev.yml` expects Postgres at `localhost:5432/aprimorar` with `myuser` / `mypassword`.
- Dev startup runs `data.sql`, which deletes and reseeds major tables. Do not expect local dev data to persist across restarts.
- Modulith boundaries matter: modules are declared in `package-info.java`, public contracts live under `...api`, implementations under `...internal`.
- If you change module boundaries or cross-module types, run `./mvnw test -Dtest=ModuleVerificationTest`; it also regenerates docs in `server/src/main/resources/docs/`.
- Use JDK 21 for verification. The project targets 21, and JaCoCo/modulith checks are not reliable on newer JDKs.

## Frontend
- Stack: React 19, Vite, TypeScript, Tailwind 4, DaisyUI, TanStack Query.
- Main frontend commands from `client/`:
  - `npm run dev`
  - `npm run build`
  - `npm run lint`
  - `npm run sync`
- `npm run sync` pulls OpenAPI from `http://localhost:8080/v3/api-docs`, so the backend must be running first.
- Generated API code lives in `client/src/kubb/` and is cleaned/regenerated by Kubb. Do not hand-edit generated files or manually create API hooks/types.
- There is no frontend test script in `client/package.json`; normal frontend verification is `npm run lint` and `npm run build`.
