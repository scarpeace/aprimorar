<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| router navigation → protected branch | unauthenticated users attempt to enter protected client routes |
| authenticated shell → logout action | trusted user action must tear down access cleanly |
| query cache → post-logout UI | stale data can persist in memory after auth is removed |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-08 | E | `client/src/App.tsx` | mitigate | Gate every protected route with the shared auth bootstrap component so anonymous users are redirected to `/login`. |
| T-01-09 | I | `client/src/components/layout/MainLayout.tsx` | mitigate | Trigger server logout and immediately redirect away from protected screens so a signed-out user cannot keep browsing cached workflows. |
| T-01-10 | T | `client/src/lib/shared/queryClient.ts` | mitigate | Clear or invalidate protected queries on logout to avoid stale protected data remaining visible after session teardown. |
</threat_model>
