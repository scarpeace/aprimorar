<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| login form → generated auth mutation | untrusted browser credentials enter the frontend auth workflow |
| browser cookies → axios client | credentialed session cookies are automatically sent on auth/API requests |
| generated Kubb hooks → handwritten auth wrapper | server contract data crosses into custom SPA state handling |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-05 | T | `client/src/lib/shared/api.ts` | mitigate | Send credentialed requests only to the configured backend origin and include Spring-compatible XSRF header/cookie support. |
| T-01-06 | I | `client/src/features/auth/pages/LoginPage.tsx` | mitigate | Avoid storing credentials in localStorage/sessionStorage; submit directly to the backend session endpoint and clear form state after submission. |
| T-01-07 | R | `client/src/features/auth/hooks/useAuthSession.ts` | mitigate | Centralize login/logout/current-user mutations so auth state invalidation is consistent and observable instead of duplicated across pages. |
</threat_model>
