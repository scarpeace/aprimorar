<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser startup → generated auth hooks | client bootstrap must configure credential and CSRF behavior before protected auth requests are sent |
| browser cookie/XSRF token → logout mutation | untrusted cross-origin request metadata must match Spring Security CSRF expectations |
| `/v1/auth/me` response → protected routing state | server-authenticated session state controls access to protected UI branches |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-11 | T | `client/src/lib/shared/api.ts` | mitigate | Force the shared axios/Kubb client to send the configured XSRF header on cross-origin cookie-authenticated mutations instead of relying on same-origin defaults. |
| T-01-12 | E | `client/src/main.tsx` | mitigate | Import the shared API bootstrap at app startup so protected auth requests never run on an unconfigured generated client. |
| T-01-13 | S | `client/src/features/auth/hooks/useAuthSession.ts` | mitigate | Derive authenticated UI state from the real `/v1/auth/me` session bootstrap rather than trusting cached login mutation data. |
</threat_model>
