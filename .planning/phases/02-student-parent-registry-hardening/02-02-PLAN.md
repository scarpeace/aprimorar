<interfaces>
Current student API shape:

```java
public record StudentRequestDTO(..., UUID parentId) {}
public record StudentResponseDTO(..., UUID parentId, Instant archivedAt, Instant createdAt, Instant updatedAt) {}
```

Replace this with explicit link DTOs so downstream frontend plans can rely on:

```java
public record StudentResponsibleRequestDTO(UUID responsibleId, String relationshipType) {}
public record StudentResponsibleResponseDTO(UUID responsibleId, String name, String contact, String cpf, String relationshipType) {}
public record StudentRequestDTO(..., List<StudentResponsibleRequestDTO> responsibles) {}
public record StudentResponseDTO(..., List<StudentResponsibleResponseDTO> responsibles, Integer responsibleCount, ...) {}
```

Keep the term `Responsável` in user-visible messages per D-10, and do not add primary/financial semantics per D-05.
</interfaces>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → student/responsável write endpoints | untrusted relationship IDs and metadata enter the API |
| parent archive/delete commands → linked students | destructive lifecycle actions can break registry integrity if unchecked |
| Springdoc/OpenAPI → generated client | contract drift can cause the SPA to submit or parse the wrong shape |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-03 | T | `StudentService#createStudent` / `updateStudent` | mitigate | Validate `responsibles[]` server-side for presence, duplicates, and real IDs before saving; never trust client-side form validation alone. |
| T-02-04 | E | `ParentService#archiveParent` / `deleteParent` | mitigate | Use repository-backed remaining-link checks so an authenticated user cannot orphan active students by archiving or deleting the last active responsável. |
| T-02-05 | I | generated Kubb contracts | mitigate | Regenerate OpenAPI + Kubb in the same plan and verify the generated DTOs expose the exact relationship fields the SPA will read and write. |
</threat_model>
